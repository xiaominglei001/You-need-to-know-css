块级元素 (block) vs. 行内元素 (inline) vs. 行内块元素 (inline-block)

1. 块级元素 (Block-level Elements)

定义： 块级元素在网页中以“块”的形式存在，它们会从新的一行开始，并占据其父元素的整个可用宽度（除非手动设置了宽度）。可以把它们想象成一个个独立的矩形盒子。

主要特点：

  • 独占一行： 每个块级元素都会另起一行，后面的元素也会被推到下一行。
  • 可设宽高： 可以自由设置 width 和 height 属性。
  • 可设边距： 所有方向的 margin 和 padding 属性都有效。
  • 默认宽度： 默认宽度是其父元素内容的100%。
  • 容器特性： 可以容纳其他块级元素和行内元素。

宽高设置与内容的关系：

  • 不设置宽高（默认）：
    • 宽度： 元素宽度默认为其父元素宽度的100%，会撑满整行。
    • 高度： 元素高度由其内部内容（文字、图片、子元素等）自动撑开。内容越多，高度越高。
  • 设置固定宽高：
    • 如果内容少于指定的宽高，元素尺寸不变，内容会在指定的空间内显示。
    • 如果内容超出指定的宽高，默认情况下内容会溢出（overflow）到元素外部，可能会破坏页面布局。你可以使用 overflow 属性（如 scroll, hidden, auto）来控制溢出行为。

常见标签： <div>, <p>, <h1>-<h6>, <ul>, <ol>, <li>, <form>, <table>, <header>, <footer> 等。

2. 行内元素 (Inline-level Elements)

定义： 行内元素（或称内联元素）不会开始新的一行，它们在文本流中“内联”排列，只占据其内容所需的宽度。它们就像段落中的单词，一个接一个地排列。

主要特点：

  • 同行显示： 与其他行内元素在同一行内显示，直到行尾空间不足才会换行。
  • 宽高无效： 直接设置 width 和 height 属性是无效的，其尺寸由内容撑开。
  • 垂直边距限制： 水平方向的 margin-left/right 和 padding-left/right 有效，但垂直方向的 margin-top/bottom 和 padding-top/bottom 不会影响布局（虽然可能会在视觉上留出空间，但不会推开上下的元素）。
  • 内容决定尺寸： 宽度和高度完全由其内部内容（如文字或图片）决定。
  • 容器限制： 通常只能容纳文本或其他行内元素（例如，你不能在一个 <span> 里放一个 <div>）。

常见标签： <span>, <a>, <img>, <strong>, <em>, <input>, <button>, <label> 等。

3. 行内块元素 (Inline-block Elements)

定义： 行内块元素是块级元素和行内元素的混合体。它既有行内元素的特性（不换行，并排显示），又有块级元素的特性（可以设置宽高和内外边距）。通过设置 display: inline-block; 来实现。

主要特点：

  • 同行显示： 像行内元素一样，可以和其他元素在同一行显示。
  • 可设宽高： 像块级元素一样，可以自由设置 width 和 height。
  • 可设边距： 像块级元素一样，所有方向的 margin 和 padding 都有效。
  • 完美结合： 它解决了“既想让元素在同一行，又想精确控制其尺寸”的常见布局问题。

应用场景： 当你需要将几个带边框、有背景、固定大小的盒子并排显示时，inline-block 是一个非常好的选择，例如导航栏按钮、产品列表项等。
